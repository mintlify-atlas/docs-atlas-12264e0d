This guide covers different methods to install and run Vespa, from quick local development setups to production deployments.

## System Requirements

Before installing Vespa, ensure your system meets these requirements:

<CardGroup cols={2}>
  <Card title="CPU Architecture" icon="microchip">
    x86_64 (AMD64) or ARM64
  </Card>
  <Card title="Memory" icon="memory">
    Minimum 4GB RAM, 8GB+ recommended
  </Card>
  <Card title="Operating System" icon="linux">
    Linux (AlmaLinux 8, Ubuntu, etc.)
  </Card>
  <Card title="Java" icon="java">
    Java 17 or later (for building applications)
  </Card>
</CardGroup>

<Note>
For production deployments, consider [Vespa Cloud](https://cloud.vespa.ai) for managed hosting with automated scaling, monitoring, and updates.
</Note>

## Installation Methods

Choose the installation method that best fits your needs:

<Tabs>
  <Tab title="Docker (Recommended)">
    Docker is the quickest way to get started with Vespa and is suitable for development and testing.
    
    ### Prerequisites
    
    - Docker Desktop or Docker Engine 20.10+
    - At least 4GB RAM allocated to Docker
    
    ### Quick Start
    
    <Steps>
      <Step title="Pull the Vespa image">
        ```bash
        docker pull vespaengine/vespa
        ```
      </Step>
      
      <Step title="Run Vespa">
        ```bash
        docker run --detach --name vespa --hostname vespa-container \
          --publish 8080:8080 --publish 19071:19071 \
          vespaengine/vespa
        ```
      </Step>
      
      <Step title="Verify installation">
        Wait for Vespa to be ready:
        
        ```bash
        docker exec vespa bash -c 'while [[ "$(curl -s -o /dev/null -w ''%{http_code}'' http://localhost:19071/state/v1/health)" != "200" ]]; do sleep 1; done'
        echo "Vespa is ready!"
        ```
      </Step>
    </Steps>
    
    ### Port Configuration
    
    The default ports are:
    
    | Port  | Service                     |
    |-------|-----------------------------|  
    | 8080  | Query and Document API      |
    | 19071 | Config server               |
    | 19050 | Node admin (internal)       |
    | 19092 | Metrics (Prometheus format) |
    
    ### Docker Compose
    
    For more complex setups, use Docker Compose:
    
    ```yaml docker-compose.yml
    version: '3'
    services:
      vespa:
        image: vespaengine/vespa
        hostname: vespa-container
        container_name: vespa
        ports:
          - "8080:8080"
          - "19071:19071"
        volumes:
          - vespa-data:/opt/vespa/var
        environment:
          - VESPA_CONFIGSERVERS=vespa-container
        healthcheck:
          test: ["CMD", "curl", "-f", "http://localhost:19071/state/v1/health"]
          interval: 10s
          timeout: 5s
          retries: 5
    
    volumes:
      vespa-data:
    ```
    
    Run with:
    ```bash
    docker-compose up -d
    ```
    
    ### Multi-node Setup with Docker
    
    For testing distributed deployments:
    
    ```yaml docker-compose-multinode.yml
    version: '3'
    services:
      config:
        image: vespaengine/vespa
        hostname: config
        command: configserver
        ports:
          - "19071:19071"
        environment:
          - VESPA_CONFIGSERVERS=config
      
      node1:
        image: vespaengine/vespa
        hostname: node1
        depends_on:
          - config
        environment:
          - VESPA_CONFIGSERVERS=config
      
      node2:
        image: vespaengine/vespa
        hostname: node2
        depends_on:
          - config
        environment:
          - VESPA_CONFIGSERVERS=config
      
      container:
        image: vespaengine/vespa
        hostname: container
        depends_on:
          - config
        ports:
          - "8080:8080"
        environment:
          - VESPA_CONFIGSERVERS=config
    ```
  </Tab>
  
  <Tab title="Package Managers">
    Install Vespa directly on Linux systems using package managers.
    
    ### RPM (Red Hat, CentOS, AlmaLinux)
    
    <Steps>
      <Step title="Add Vespa repository">
        ```bash
        sudo dnf config-manager --add-repo \
          https://copr.fedorainfracloud.org/coprs/g/vespa/vespa/repo/epel-8/group_vespa-vespa-epel-8.repo
        ```
      </Step>
      
      <Step title="Install Vespa">
        ```bash
        sudo dnf install vespa
        ```
      </Step>
      
      <Step title="Start Vespa services">
        ```bash
        sudo systemctl start vespa-configserver
        sudo systemctl start vespa
        ```
      </Step>
    </Steps>
    
    ### DEB (Ubuntu, Debian)
    
    <Steps>
      <Step title="Add Vespa repository">
        ```bash
        sudo apt-get install -y apt-transport-https ca-certificates curl gnupg
        curl -fsSL https://vespa.ai/keys/vespa.gpg | sudo gpg --dearmor -o /usr/share/keyrings/vespa-archive-keyring.gpg
        echo "deb [signed-by=/usr/share/keyrings/vespa-archive-keyring.gpg] https://vespa.ai/repo/vespa ubuntu main" | sudo tee /etc/apt/sources.list.d/vespa.list
        ```
      </Step>
      
      <Step title="Install Vespa">
        ```bash
        sudo apt-get update
        sudo apt-get install -y vespa
        ```
      </Step>
      
      <Step title="Start Vespa services">
        ```bash
        sudo systemctl start vespa-configserver
        sudo systemctl start vespa
        ```
      </Step>
    </Steps>
    
    ### Verify Installation
    
    ```bash
    # Check service status
    sudo systemctl status vespa
    
    # Check config server
    curl http://localhost:19071/state/v1/health
    ```
  </Tab>
  
  <Tab title="Build from Source">
    Build Vespa from source for development or custom deployments.
    
    ### Prerequisites
    
    Building Vespa requires:
    
    - Java 17 or later
    - Maven 3.8+
    - C++ compiler (GCC 12+ or Clang)
    - CMake 3.23+
    - AlmaLinux 8 (recommended for full build)
    
    <Warning>
    Building the complete Vespa codebase (C++ and Java) is only supported on AlmaLinux 8. The Java modules can be built on any platform with Java 17+ and Maven 3.8+.
    </Warning>
    
    ### Java-only Build
    
    For application development and testing:
    
    <Steps>
      <Step title="Clone the repository">
        ```bash
        git clone https://github.com/vespa-engine/vespa.git
        cd vespa
        ```
      </Step>
      
      <Step title="Set up environment">
        ```bash
        export MAVEN_OPTS="-Xms128m -Xmx1024m"
        ```
      </Step>
      
      <Step title="Build Java modules">
        ```bash
        ./bootstrap.sh java
        mvn install --threads 1C
        ```
        
        This builds all Java modules using one thread per CPU core.
      </Step>
    </Steps>
    
    ### Full Build (C++ and Java)
    
    For complete Vespa development:
    
    <Steps>
      <Step title="Set up development environment">
        Use the Docker development image:
        
        ```bash
        docker pull vespaengine/vespa-dev-almalinux-8:latest
        docker run -ti -v $(pwd):/vespa --entrypoint /bin/bash \
          vespaengine/vespa-dev-almalinux-8:latest
        ```
        
        See the [development guide](https://github.com/vespa-engine/docker-image-dev#vespa-development-on-almalinux-8) for details.
      </Step>
      
      <Step title="Build Vespa">
        Inside the container:
        
        ```bash
        cd /vespa
        ./bootstrap.sh full
        mvn install --threads 1C
        ```
      </Step>
    </Steps>
    
    ### macOS Development Setup
    
    For Java module development on macOS:
    
    <Steps>
      <Step title="Install dependencies">
        ```bash
        brew install jenv mvnvm openjdk@17
        ```
      </Step>
      
      <Step title="Configure environment">
        ```bash
        # Link OpenJDK
        sudo ln -sfn /opt/homebrew/opt/openjdk@17/libexec/openjdk.jdk \
          /Library/Java/JavaVirtualMachines/openjdk-17.jdk
        
        # Set up jEnv (for zsh)
        echo 'export PATH="$HOME/.jenv/bin:$PATH"' >> ~/.zshrc
        echo 'eval "$(jenv init -)"' >> ~/.zshrc
        eval "$(jenv init -)"
        jenv enable-plugin export
        jenv add $(/usr/libexec/java_home -v 17)
        ```
      </Step>
      
      <Step title="On ARM Macs, install Rosetta">
        Required for gRPC compatibility:
        
        ```bash
        softwareupdate --install-rosetta
        ```
      </Step>
      
      <Step title="Verify setup">
        ```bash
        mvn -v
        ```
        
        Should show Java 17 and Maven 3.8+.
      </Step>
    </Steps>
    
    ### Repository Structure
    
    The Vespa codebase contains ~1.7 million lines of code across 150+ modules:
    
    - **[container-search](https://github.com/vespa-engine/vespa/tree/master/container-search)**: Query processing framework
    - **[searchcore](https://github.com/vespa-engine/vespa/tree/master/searchcore)**: Core indexing and matching (C++)
    - **[searchlib](https://github.com/vespa-engine/vespa/tree/master/searchlib)**: Ranking and feature libraries (C++)
    - **[configserver](https://github.com/vespa-engine/vespa/tree/master/configserver)**: Configuration management
    - **[document](https://github.com/vespa-engine/vespa/tree/master/document)**: Document model (Java and C++)
    
    See [Code-map.md](https://github.com/vespa-engine/vespa/blob/master/Code-map.md) for a complete overview.
  </Tab>

## Vespa CLI

The Vespa CLI is a command-line tool for managing Vespa applications.

### Installation

<CodeGroup>

```bash macOS
brew install vespa-cli
```

```bash Linux
curl -fsSL https://dl.vespa.ai/vespa-cli/install.sh | sh
```

```bash Windows (WSL)
curl -fsSL https://dl.vespa.ai/vespa-cli/install.sh | sh
```

```bash Go
go install github.com/vespa-engine/vespa/client/go/cmd/vespa@latest
```

</CodeGroup>

### Common Commands

```bash
# Deploy an application
vespa deploy

# Query
vespa query 'select * from music where true'

# Feed a document
vespa document put id:music:music::1 doc.json

# Check status
vespa status

# View logs
vespa log
```

See the [Vespa CLI reference](https://docs.vespa.ai/en/reference/clients/vespa-cli.html) for complete documentation.

## Production Considerations

When deploying Vespa in production, consider:

### Hardware Requirements

<CardGroup cols={2}>
  <Card title="Config Server" icon="server">
    - 2+ CPU cores
    - 4GB+ RAM
    - 50GB+ disk
    - Odd number (1, 3, or 5)
  </Card>
  <Card title="Container Nodes" icon="layer-group">
    - 4+ CPU cores
    - 8GB+ RAM
    - Scales horizontally
  </Card>
  <Card title="Content Nodes" icon="database">
    - 8+ CPU cores
    - 16GB+ RAM
    - Fast SSD storage
    - Size based on data volume
  </Card>
  <Card title="Network" icon="network-wired">
    - Low latency between nodes
    - 1Gbps+ bandwidth
    - Reliable connections
  </Card>
</CardGroup>

### Multi-node Setup

For production, deploy Vespa across multiple nodes:

- **Config server cluster**: 3 nodes for high availability
- **Container cluster**: Scale based on query load
- **Content cluster**: Scale based on data size and redundancy needs

Example `services.xml` for multi-node:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<services version="1.0">
    <admin version="2.0">
        <adminserver hostalias="admin0"/>
        <configservers>
            <configserver hostalias="config0"/>
            <configserver hostalias="config1"/>
            <configserver hostalias="config2"/>
        </configservers>
    </admin>
    
    <container id="default" version="1.0">
        <search/>
        <document-api/>
        <nodes>
            <node hostalias="container0"/>
            <node hostalias="container1"/>
            <node hostalias="container2"/>
        </nodes>
    </container>
    
    <content id="content" version="1.0">
        <redundancy>2</redundancy>
        <documents>
            <document type="music" mode="index"/>
        </documents>
        <nodes>
            <node hostalias="content0" distribution-key="0"/>
            <node hostalias="content1" distribution-key="1"/>
            <node hostalias="content2" distribution-key="2"/>
        </nodes>
    </content>
</services>
```

See [multinode systems](https://docs.vespa.ai/en/multinode-systems.html) for complete documentation.

### Monitoring

Vespa exposes metrics in Prometheus format:

```bash
curl http://localhost:19092/prometheus/v1/values
```

Key metrics to monitor:

- Query latency (p50, p95, p99)
- Query throughput
- Feed throughput
- Disk usage
- Memory usage
- CPU usage

### Security

For production deployments:

- Enable TLS for all endpoints
- Configure authentication and authorization
- Use firewall rules to restrict access
- Regular security updates

See [securing Vespa](https://docs.vespa.ai/en/securing-your-vespa-installation.html) for details.

## Troubleshooting

<AccordionGroup>
  <Accordion title="Container won't start">
    Check Docker resource limits:
    
    ```bash
    docker stats vespa
    ```
    
    Vespa needs at least 4GB RAM. Increase Docker's memory limit if needed.
  </Accordion>
  
  <Accordion title="Config server not responding">
    Verify the config server is running:
    
    ```bash
    docker exec vespa systemctl status vespa-configserver
    ```
    
    Check logs:
    ```bash
    docker exec vespa vespa-logfmt | grep configserver
    ```
  </Accordion>
  
  <Accordion title="Build fails on macOS">
    For ARM Macs, ensure Rosetta is installed:
    
    ```bash
    softwareupdate --install-rosetta
    ```
    
    Verify Java version:
    ```bash
    java -version  # Should be 17+
    mvn -v         # Should use Java 17+
    ```
  </Accordion>
  
  <Accordion title="Out of memory errors">
    Increase JVM memory for Maven:
    
    ```bash
    export MAVEN_OPTS="-Xms256m -Xmx2048m"
    ```
    
    For Docker, increase container memory limits.
  </Accordion>
</AccordionGroup>
